# RV64IM_Zicsr_Zifencei 五级流水线虚拟仿真实验软件 — 使用说明书

## 一、软件概述

本软件是 **RV64IM_Zicsr_Zifencei 五级流水线虚拟仿真实验平台**，用于教学和实验目的，帮助用户直观地理解 RISC-V 处理器的五级流水线（IF → ID → EX → MEM → WB）工作原理。

软件提供两种核心使用模式：

| 模式 | 适用场景 | 主要功能 |
|------|----------|----------|
| **自由编程模式** | 学习流水线执行过程、观察内部数据流 | 编写汇编代码 → 编译加载 → 单步/连续运行 → 观察流水线细节 |
| **差分测试模式** | 学习控制信号（RegWrite、ALUSrc 等） | 选择教学场景 → 加载测试用例 → 运行并对比控制信号 |

## 二、界面布局说明

软件界面分为以下区域：

```
┌─────────────────────────────────────────────────────────────┐
│                    顶部控制栏（运行/暂停/重置等按钮）          │
├──────────┬──────────────────────────────┬───────────────────┤
│ 左侧面板  │      中央流水线可视化区域      │     右侧面板       │
│ (差分测试面板  │   （五级流水线 + 数据连线  │   (代码编辑 /     │
│ 或流水线信息)  │    + 模块点击查看详情）   │    CPU状态信息)    │
├──────────┴──────────────────────────────┴───────────────────┤
│                      底部统计栏                              │
└─────────────────────────────────────────────────────────────┘
```

- **顶部控制栏**：包含运行控制按钮（下一 clk、运行、暂停、重置）和速度调节
- **中央区域**：流水线结构图，展示 IF/ID/EX/MEM/WB 五个阶段及各模块间的数据连线
- **左侧面板**：可切换显示「流水线信息」或其他面板
- **右侧面板**：可切换显示「代码编辑面板」或「CPU 状态信息」面板
- **底部统计栏**：显示运行统计数据

### 流水线可视化区域

中央区域展示了完整的五级流水线结构，包括：

- **五大功能单元**：Fetch Unit（取指）、Decode Unit（译码）、Execute Unit（执行）、Memory Unit（访存）、WriteBack Unit（写回）
- **四个流水线寄存器**：IF/ID、ID/EX、EX/MEM、MEM/WB
- **辅助模块**：Register File（寄存器堆）、instMEM（指令内存）、Data Memory（数据内存）、Control Unit（控制单元）
- **数据连线**：模块之间的数据通路，**活跃的数据流会高亮显示**（蓝色=数据，绿色=地址，红色=控制信号）

> **交互提示**：点击任意模块（如 Register File、ALU、Control Unit、各流水线寄存器）可弹出详情窗口，查看该模块的内部数据。

## 三、模式一：自由编程模式 — 编写代码观察流水线

此模式适合从零开始学习流水线的工作流程，通过自己编写汇编代码，逐步观察指令在流水线中的执行细节。

### 步骤 1：打开代码编辑器

在右侧面板中找到 **「汇编代码编辑器」** 面板。

### 步骤 2：编写 RISC-V 汇编代码

在代码编辑区编写你的汇编程序。代码必须遵循以下格式要求：

```asm
# 必须包含 .text 段 - 表示代码段
# 必须包含 .globl _start - 声明程序入口点
# 代码写在 _start: 标签之后
# 必须以 ebreak 结尾 - 停止程序运行
.text
.globl _start
_start:
    addi x1, x0, 10       # x1 = 10
    addi x2, x0, 20       # x2 = 20
    add  x3, x1, x2       # x3 = x1 + x2 = 30
    ebreak                 # 停止
```

**代码规范要点**：
- 必须包含 `.text` 段声明
- 必须包含 `.globl _start` 入口点声明
- 有效指令写在 `_start:` 标签之后
- 程序末尾必须以 `ebreak` 指令结束

**支持的指令集**：RV64IM（整数运算 + 乘除法）+ Zicsr（CSR 指令）+ Zifencei（ fence.i）

也可以通过以下方式加载已有代码：
- 点击 **「上传.S文件」** 按钮，选择本地的 `.S` / `.s` / `.asm` 汇编文件
- 直接将文件 **拖拽** 到编辑器区域

### 步骤 3：编译并加载

点击 **「编译运行」** 按钮：

- **编译成功**：底部显示绿色提示 "编译成功！程序已加载"，此时可以开始运行
- **编译失败**：底部显示红色错误信息，请根据提示修改代码后重新编译

### 步骤 4：运行与观察

编译成功后，使用顶部控制栏的按钮控制程序执行：

| 按钮 | 功能 | 说明 |
|------|------|------|
| **下一 clk** | 单步执行一个时钟周期 | 推荐初学者使用，便于逐周期观察 |
| **运行** | 自动连续执行 | 可设置自动运行速度（0.3s ~ 5s） |
| **暂停** | 暂停自动运行 | 暂停后可切换为单步模式 |
| **重置** | 重置整个流水线 | 清空所有状态，回到初始状态 |

**速度调节**：点击时钟图标旁的下拉框，可选择每个时钟周期的间隔时间（0.3秒 / 0.5秒 / 1秒 / 2秒 / 5秒）。

### 步骤 5：观察流水线内部细节

运行过程中，可以通过以下方式深入观察流水线：

#### （1）观察数据流高亮

流水线图上的连线会根据当前执行的指令**动态高亮**：
- **蓝色高亮**：正在传输的数据信号
- **绿色高亮**：地址相关信号（PC、存储地址等）
- **红色高亮**：控制信号（allow_to_go、flush 等）

#### （2）查看模块详情

点击流水线上的模块，弹出对应详情窗口：

| 点击模块 | 弹出内容 |
|----------|----------|
| Register File | 32 个通用寄存器（x0~x31）的当前值 |
| Execute Unit / ALU | ALU 的两个操作数、运算类型、运算结果 |
| Control Unit | 所有控制信号的当前值 |
| Fetch Unit | IF/ID 流水线寄存器内容（PC、指令、valid 等） |
| Decode Unit | ID/EX 流水线寄存器内容 |
| Memory Unit | EX/MEM 流水线寄存器内容 |
| WriteBack Unit | MEM/WB 流水线寄存器内容 |

#### （3）查看 CPU 状态信息

在右侧面板中切换到 **「CPU 状态信息」** 面板，可查看：
- **基本信息**：当前周期数、PC 值
- **活跃信号列表**：所有当前活跃的控制信号和数据信号及其值
- **通用寄存器堆**：32 个寄存器的实时值（有变化的寄存器会高亮显示）

## 四、模式二：差分测试模式 — 测试控制信号

此模式专为学习 **RISC-V 控制信号** 设计，通过预设的教学场景和测试用例，帮助理解不同指令对应的控制信号（RegWrite、ALUSrc、MemRead、MemWrite、Branch）是如何产生的。

### 步骤 1：打开差分测试面板

在左侧面板中找到 **「差分测试」** 面板。

### 步骤 2：选择教学场景

软件提供 **4 个渐进式教学场景** + **1 个自由模式**：

| 场景 | 名称 | 学习目标 | 涉及的控制信号 |
|------|------|----------|----------------|
| 场景 1 | 寄存器写 | 区分哪些指令会写寄存器，哪些不会 | RegWrite |
| 场景 2 | 操作数选择 | 理解 ALU 如何选择操作数（立即数 vs 寄存器） | RegWrite + ALUSrc |
| 场景 3 | 内存访问 | 理解 Load/Store 指令的内存读写控制 | RegWrite + ALUSrc + MemRead + MemWrite |
| 场景 4 | 分支控制 | 理解分支跳转对流水线的影响 | 全部 5 个信号 |
| 自由模式 | — | 自定义想要观察的信号组合 | 用户自选 |

**操作方法**：点击场景名称即可选中（单选）。

### 步骤 3：（可选）选择测试用例

选中场景后，可以在 **「测试用例」** 区域选择配套的测试程序：

> 如果不选择测试用例，也可以使用自己在代码编辑器中编译的程序进行差分测试，但要**先加载程序，再配置差分测试选项**。

### 步骤 4：确认配置或加载运行

有两种启动方式：

#### 方式 A：使用已编译的自定义代码
1. 先在右侧代码编辑器中编写并**编译成功**代码
2. 在差分测试面板中选择场景
3. 点击 **「确认配置」** 按钮
4. 使用顶部控制栏的「运行」/「下一 clk」按钮执行

#### 方式 B：直接加载测试用例
1. 选择场景和配套的测试用例（推荐选 ELF 测试）
2. 点击 **「加载并运行」** 按钮
3. 测试程序会自动加载并开始差分测试

### 步骤 5：观察差分测试结果

运行过程中，系统会将**你的实现（DUT）** 与 **参考实现（Ref）** 的控制信号进行实时对比：

- 当检测到控制信号不一致时，系统会**自动暂停**并弹出 **差分结果窗口**
- 窗口中会清晰展示：
  - 当前指令是什么
  - 你的值 vs 期望的正确值

### 步骤 6：禁用差分测试

如需退出差分测试模式，点击 **「禁用」** 按钮即可恢复到普通运行模式。

## 五、各教学场景详解

### 场景 1：RegWrite（寄存器写入使能）

**学习目标**：理解哪些指令需要将结果写回寄存器堆。

**关键概念**：
- `RegWrite = 1`：指令执行结果需要写入寄存器（如 ADD、ADDI、LW 等）
- `RegWrite = 0`：指令不需要写寄存器（如 SW、BEQ、NOP 等）

**典型指令对比**：
| 指令 | RegWrite | 说明 |
|------|----------|------|
| ADD x3, x1, x2 | 1 | 算术运算结果写入 x3 |
| ADDI x1, x0, 10 | 1 | 立即数加法结果写入 x1 |
| SW x1, 0(x2) | 0 | 存储指令，不写寄存器 |
| BEQ x1, x2, target | 0 | 分支指令，不写寄存器 |

### 场景 2：ALUSrc（ALU 操作数选择）

**学习目标**：理解 ALU 第二个操作数的来源选择。

**关键概念**：
- `ALUSrc = 0`：第二个操作数来自寄存器（rs2）（如 ADD、SUB、AND 等 R 型指令）
- `ALUSrc = 1`：第二个操作数来自立即数（如 ADDI、ANDI、LW 等 I 型指令）

**典型指令对比**：
| 指令 | ALUSrc | ALU 操作数2 来源 |
|------|--------|------------------|
| ADD x3, x1, x2 | 0 | rs2（x2 的值） |
| ADDI x1, x0, 10 | 1 | 立即数 10 |
| LW x3, 4(x1) | 1 | 偏移量立即数 4 |

### 场景 3：MemRead / MemWrite（内存访问控制）

**学习目标**：理解 Load 和 Store 指令如何控制数据内存的读写。

**关键概念**：
- `MemRead = 1`：从数据内存读取数据（Load 指令：LB、LH、LW、LBU、LHU 等）
- `MemWrite = 1`：向数据内存写入数据（Store 指令：SB、SH、SW 等）
- 其余时候两者均为 0

### 场景 4：Branch（分支控制）

**学习目标**：理解分支指令对流水线的影响（flush 机制）。

**关键概念**：
- `Branch = 1`：当前是分支指令（BEQ、BNE、BLT、BGE、BLTU、BGEU）
- 分支命中时会产生 **flush 信号**，清空后续流水线阶段的无效指令
- PC 会被更新为分支目标地址

## 七、常见问题

### Q1：编译失败怎么办？
A：检查以下几点：
- 代码是否包含 `.text` 和 `.globl _start`
- 指令拼写是否正确
- 寄存器名是否合法（x0 ~ x31）
- 是否以 `ebreak` 结尾

### Q2：运行时程序没有反应？
A：确认代码已经**编译成功**（底部出现绿色成功提示），再点击运行按钮。

### Q3：差分测试时一直提示"请先加载程序"？
A：需要先在代码编辑器中编译代码，或者在选择场景后在测试用例中选择一个测试程序。

### Q4：如何查看某个寄存器的值？
A：两种方式：
- 右侧 CPU 状态信息面板中的「通用寄存器堆」区域
- 点击流水线图中的 Register File 模块

### Q5：流水线图太大/太小怎么看？
A：使用鼠标滚轮缩放，拖拽平移，双击重置视图。
