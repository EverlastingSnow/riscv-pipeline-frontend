## 软件简介

RISC-V 五级流水线虚拟仿真实验软件是一款基于 Vue 3 + TypeScript 的交互式流水线可视化工具，用于学习和理解 RISC-V 架构的五级流水线执行过程。

### 技术栈

| 技术 | 说明 |
|------|------|
| Vue 3 | 前端框架 |
| TypeScript | 类型安全 |
| Tailwind CSS | 样式框架 |
| Pinia | 状态管理 |
| Lucide Icons | 图标库 |
| Vite | 构建工具 |

### 五级流水线阶段

| 阶段 | 名称 | 说明 |
|------|------|------|
| IF | 取指 (Instruction Fetch) | 从指令存储器读取指令 |
| ID | 译码 (Instruction Decode) | 解析指令并读取寄存器 |
| EX | 执行 (Execute) | ALU 执行运算或计算地址 |
| MEM | 访存 (Memory Access) | 访问数据存储器 |
| WB | 写回 (Write Back) | 将结果写回寄存器 |

## 主要功能

### 3.1 流水线可视化

- **模块展示**：显示 Fetch Unit、Decode Unit、Execute Unit、Memory Unit、WriteBack Unit 五个核心模块
- **流水线寄存器**：显示 IF/ID、ID/EX、EX/MEM、MEM/WB 四个流水线寄存器
- **辅助模块**：显示寄存器文件(Register File)、指令存储器(instMEM)、数据存储器(Data Memory)、控制单元(Control Unit)
- **数据流高亮**：动态显示当前活跃的数据流和控制信号
- **连接线标注**：显示各模块间的信号名称(PC、inst、valid、src1_raddr 等)

### 3.2 控制功能

| 按钮 | 功能 | 说明 |
|------|------|------|
| 下一 clk | 单步执行 | 执行一个时钟周期 |
| 运行 | 自动运行 | 连续执行流水线，可设置间隔时间 |
| 暂停 | 暂停运行 | 暂停自动执行 |
| 重置 | 重置流水线 | 恢复初始状态 |
| 差分测试 | 差分测试配置 | 配置并启用差分测试模式 |

### 3.3 代码编辑与编译

- **内置代码编辑器**：支持编写 RISC-V 汇编代码
- **文件上传**：支持上传 .S 汇编文件
- **在线编译**：将汇编代码编译为 ELF 格式
- **代码模板**：提供默认示例代码

### 3.4 状态监控

- **CPU 状态信息**：显示当前周期、PC 值
- **寄存器状态**：32 个通用寄存器的值 (x0-x31)
- **ALU 状态**：ALU 输入输出数据
- **控制信号**：各模块的控制信号状态
- **流水线寄存器**：各级流水线的内容

### 3.5 差分测试

支持多种教学场景的差分测试：
- 场景 1: 寄存器写 (RegWrite)
- 场景 2: 操作数选择 (RegWrite + ALUSrc)
- 场景 3: 内存访问 (RegWrite + ALUSrc + MemRead + MemWrite)
- 场景 4: 分支控制 (RegWrite + ALUSrc + MemRead + MemWrite + Branch)
- 自由模式：自定义所有信号

## 操作指南

### 4.1 启动软件

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问显示的地址（通常为 http://localhost:5173）即可使用。

### 4.2 基本操作流程

#### 步骤 1：编写或加载代码

在代码编辑器中编写 RISC-V 汇编代码，或点击上传按钮加载 .S 文件。

示例代码：
```assembly
.text
.globl _start
_start:
    addi x1, x0, 10
    addi x2, x0, 20
    add x3, x1, x2
    ebreak
```

#### 步骤 2：编译代码

点击"编译运行"按钮，系统将代码发送至后端编译。

- 编译成功：自动加载 ELF 到流水线
- 编译失败：显示错误信息

#### 步骤 3：执行流水线

| 操作 | 按钮 | 说明 |
|------|------|------|
| 单步执行 | 下一 clk | 每点击执行一个时钟周期 |
| 自动运行 | 运行 | 连续执行，可设置间隔 (0.3s-5s) |
| 暂停 | 暂停 | 暂停自动执行 |
| 重置 | 重置 | 重新开始 |

#### 步骤 4：查看状态

- **点击模块**：弹出详情窗口（寄存器堆、ALU、控制信号等）
- **查看流水线寄存器**：底部浮动面板显示各级内容
- **右侧状态面板**：显示 CPU 状态、寄存器值、控制信号等

### 4.3 差分测试操作

1. 点击"差分测试"按钮打开配置窗口
2. 选择测试场景
3. 选择测试用例
4. 点击"应用配置"
5. 使用"下一 clk"或"运行"执行测试
6. 查看测试结果

### 4.4 模块交互

| 操作 | 效果 |
|------|------|
| 单击模块 | 查看模块详情 |
| 单击流水线寄存器 | 查看寄存器内容 |
| 双击模块 | 展开/折叠详情 |
| 拖拽模块 | 调整布局（需开启编辑模式） |

## 常见问题

### Q1：软件无法启动？

检查端口是否被占用，或尝试更换端口：
```bash
npm run dev -- --port 3000
```

### Q2：代码编译失败？

- 确保代码包含 `.text` 段和 `.globl _start`
- 代码必须以 `ebreak` 结尾
- 检查汇编语法是否正确

### Q3：流水线不执行？

1. 确认代码已编译成功
2. 检查是否已按"编译运行"加载 ELF
3. 查看后端服务是否正常运行

### Q4：差分测试无法启用？

- 确认已选择测试场景
- 确认已选择测试用例
- 检查后端是否支持差分测试

### Q5：如何联系支持？

如遇问题，请联系实验指导教师或技术支持团队。
